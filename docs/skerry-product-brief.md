# Skerry — продуктовый базис

> Опенсорсный кроссплатформенный SSH-клиент, функциональный аналог коммерческих SSH-клиентов,
> с единым ядром под капотом. Зафиксировано 2026-06-11 по итогам сессии вопросов-ответов.

## Принятые решения

| Вопрос | Решение |
|---|---|
| Стек / единое ядро | **Kotlin Multiplatform**: общее ядро (SSH, крипто, хранилище, sync) |
| UI-слой | **Compose Multiplatform везде** — Android, iOS, Desktop (JVM), ~90% общего кода |
| Платформы | **Все сразу, паритет с первого релиза**: Linux / Windows / macOS / Android / iOS+iPadOS |
| Объём MVP | **SSH + SFTP + port forwarding** (сниппеты, Mosh, Telnet, serial — позже) |
| Синхронизация | **Self-hosted sync-сервер** (Docker), E2E-шифрование мастер-паролем, сервер видит только шифроблобы — модель Vaultwarden |
| AI-ассистент | **В ядре, но после MVP (Phase 2)**: интерфейсы и политики (Strict/Balanced/Permissive/Off) проектируются сразу, реализация — после базового SSH/SFTP |
| Лицензия | **GPL-3.0 клиенты + AGPL-3.0 sync-сервер** |
| Дистрибуция: Linux | **Flatpak/Flathub — основной канал** (в манифесте `--socket=ssh-auth` для ssh-agent) + `.deb`/`.rpm` из jpackage в GitHub Releases |
| Дистрибуция: Windows | **MSI (jpackage) + портативный ZIP** (для машин без прав администратора); winget — позже как дешёвое дополнение |
| Дистрибуция: macOS | **DMG (universal arm64+x64) без нотаризации на старте** — пользователь обходит Gatekeeper вручную; нотаризация и Homebrew cask, когда появятся пользователи |
| Обновления desktop | **Через каналы** (Flathub/пакеты); приложение лишь проверяет GitHub Releases API и показывает «доступна новая версия», собственного автоапдейтера нет |

## Структура кодовой базы (целевая)

```
skerry/
├─ shared/            # ядро: общая Kotlin-логика
│  ├─ ssh/            # SSH-транспорт, каналы, port forwarding
│  ├─ sftp/           # файловые операции, очередь передач
│  ├─ vault/          # хранилище хостов/ключей, шифрование (master password → dataKey)
│  ├─ sync/           # клиент E2E-синхронизации
│  ├─ terminal/       # эмулятор терминала (VT100/xterm state machine)
│  └─ ai/             # интерфейсы AI-провайдеров и политик (реализация — Phase 2)
├─ composeApp/        # ВЕСЬ UI один раз (Compose Multiplatform)
│  ├─ commonMain/     # экраны: hosts, terminal, sftp, keys, settings, lock
│  ├─ androidMain/
│  ├─ iosMain/
│  └─ desktopMain/    # JVM: Linux/Windows/macOS
├─ server/            # self-hosted sync-сервер (Ktor), AGPL-3.0
└─ docs/              # прототипы и документы
```

|---|---|
| `skerry-prototype.html` | Desktop: titlebar c вкладками сессий, sidebar c группами хостов, терминал, AI-карточки и политики, lock-экран, настройки |
| `skerry-mobile-prototype.html` | Mobile (рамка телефона): lock, список хостов, терминал с панелью спецклавиш и сниппетами, SFTP с прогрессом передач, keychain (ключи + known hosts), настройки, bottom-sheet нового подключения |
| `skerry-tablet-prototype.html` | Tablet/iPad (ландшафт 11″): «почти desktop»-режим — постоянный скрываемый сайдбар, вкладки сессий, split-view двух терминалов, SFTP в двухпанельном режиме (iPad ↔ сервер) с очередью передач, компактная однорядная тач-панель спецклавиш (авто-скрытие при аппаратной клавиатуре), настройки sync в поповере |

| `skerry-sync-design.md` | Проект протокола sync: иерархия ключей (Argon2id → masterKey → authKey/dataKey), модель данных VaultRecord, REST/WS API, LWW-разрешение конфликтов, паринг устройств, модель угроз |

Дизайн-токены общие для обоих прототипов (палитра «night sea»: cyan — primary,
amber — только AI/lighthouse-моменты, JetBrains Mono для всего терминального).
Эти токены переносятся в Compose-тему как единый источник правды.

## Фазы

1. **Phase 1 — MVP**: ядро SSH/SFTP/port forwarding, менеджер хостов и ключей,
   терминал, мастер-пароль + биометрия, локальное зашифрованное хранилище.
   Релиз на всех 5 платформах одновременно.
2. **Phase 2**: self-hosted sync-сервер, сниппеты с переменными, AI-ассистент
   (BYOK / свой endpoint, политики per-host), jump hosts, agent forwarding.
3. **Phase 3**: Mosh, Telnet, serial, autocomplete, темы сообщества,
   опциональная локальная AI-модель на desktop.

## Принципы

- **Local-first**: всё работает без сервера; sync — опциональная надстройка.
- **Zero-knowledge**: мастер-пароль не покидает устройство; сервер хранит только ciphertext.
- **AI under policy**: любые AI-функции подчиняются per-host политике; вывод с сервера
  считается недоверенным источником (prompt injection) — подтверждение перед выполнением.
- **Паритет платформ**: фича не считается готовой, пока не работает на всех платформах.
